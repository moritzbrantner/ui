# Define Pop and Pulse theme semantics

Pop is the polished public expression of Studio rather than a broad playful consumer skin, and Pulse is the kinetic interaction theme rather than a realtime/status theme. We made this split because the old labels made Pop overlap with generic consumer products and made Pulse sound tied to live data, while the intended design language is Studio-adjacent creator polish for Pop and choreographed open, close, expand, collapse, and selection movement for Pulse.

The public theme profile surface vocabulary now includes `kinetic` for Pulse. Pop keeps a consumer-facing surface because it is still aimed at public creator experiences, but its label, description, and recommended use cases should point back to Studio rather than generic creator tooling.
